# Список желаний (Wishlist)

Приложение для управления списком желаний с регистрацией, авторизацией и возможностью добавлять подарки с изображениями.

## Технологии

### Backend
- **Node.js** + **Express** - серверная часть
- **MongoDB** + **Mongoose** - база данных
- **JWT** - аутентификация
- **bcryptjs** - хеширование паролей
- **multer** - загрузка файлов

### Frontend
- **Vue 3** - фреймворк
- **Vue Router** - маршрутизация
- **Axios** - HTTP запросы
- **Vite** - сборщик

## Установка и запуск

### 1. Настройка базы данных MongoDB

У вас есть два варианта:

#### Вариант А: Локальная установка MongoDB
1. Скачайте и установите MongoDB: https://www.mongodb.com/try/download/community
2. Запустите MongoDB сервис
3. MongoDB будет доступен на `mongodb://localhost:27017`

#### Вариант Б: MongoDB Atlas (облачная база данных, бесплатно)
1. Зарегистрируйтесь на https://www.mongodb.com/cloud/atlas
2. Создайте бесплатный кластер
3. Получите строку подключения (Connection String)

### 2. Настройка Backend

```bash
# Перейдите в папку backend
cd backend

# Установите зависимости
npm install

# Создайте файл .env (скопируйте .env.example если есть)
# Добавьте в него:
MONGODB_URI=mongodb://localhost:27017/wishlist
# Или для MongoDB Atlas:
# MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/wishlist

JWT_SECRET=your-super-secret-jwt-key-change-this-in-production
PORT=5000
NODE_ENV=development

# Запустите сервер
npm run dev
```

Сервер будет доступен на http://localhost:5000

### 3. Настройка Frontend

```bash
# Откройте новый терминал и перейдите в папку client
cd client

# Установите зависимости
npm install

# Запустите dev сервер
npm run dev
```

Фронтенд будет доступен на http://localhost:5173 (или другом порту, который укажет Vite)

## Использование

1. Откройте браузер и перейдите на адрес фронтенда (обычно http://localhost:5173)
2. Зарегистрируйте новый аккаунт (логин от 3 до 20 символов, пароль минимум 6 символов)
3. Войдите в систему
4. Добавляйте подарки с названием и изображением (опционально)
5. Редактируйте или удаляйте подарки по необходимости

## API Endpoints

### Авторизация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход
- `GET /api/auth/me` - Получить информацию о текущем пользователе (требует токен)

### Подарки (требуют авторизации)
- `GET /api/wishes` - Получить все подарки текущего пользователя
- `POST /api/wishes` - Создать новый подарок (multipart/form-data: title, image)
- `PUT /api/wishes/:id` - Обновить подарок (multipart/form-data: title, image)
- `DELETE /api/wishes/:id` - Удалить подарок

## Объяснение основных концепций

### Backend

**Express** - это веб-фреймворк для Node.js, который обрабатывает HTTP запросы.

**Mongoose** - это библиотека для работы с MongoDB. Она позволяет определять схемы (структуру данных) и работать с базой данных через JavaScript объекты.

**JWT (JSON Web Token)** - это способ безопасной передачи информации между клиентом и сервером. После успешного входа, сервер выдает токен, который клиент сохраняет и отправляет с каждым запросом.

**bcryptjs** - хеширует пароли перед сохранением в базу данных, чтобы даже если база будет скомпрометирована, пароли нельзя будет прочитать.

**multer** - обрабатывает загрузку файлов (изображений) на сервер.

### Frontend

**Vue 3** - это прогрессивный JavaScript фреймворк для создания пользовательских интерфейсов. Использует реактивность - когда данные меняются, интерфейс автоматически обновляется.

**Vue Router** - управляет навигацией между страницами в одностраничном приложении (SPA).

**Axios** - библиотека для выполнения HTTP запросов к серверу.

**Vite** - быстрый сборщик и dev-сервер для разработки.

## Безопасность

- Пароли хешируются перед сохранением
- JWT токены используются для аутентификации
- Пользователи могут видеть и редактировать только свои подарки
- Валидация данных на клиенте и сервере
- Ограничение размера загружаемых файлов (5MB)
